## 摘要

Andrej Karpathy 认为 LLM 输出应从 Markdown 演进到 HTML，最终走向交互式神经媒体。视觉是大脑带宽最高的输入通道（约占皮层处理的 1/3），AI 输出应越来越多地利用视觉丰富性。一个实用技巧：在提示词末尾加上 "structure your response as HTML" 即可获得立竿见影的效果提升。

---

## 要点

- **输出格式递进**：纯文本 → Markdown → HTML → 交互式神经视频，每一步都提升了向人脑传递信息的带宽。
- **视觉是主要输出通道**：人脑约 1/3 专门处理视觉信息，AI 应充分利用这条"10 车道高速公路"。
- **HTML 是实际可行的下一步**：比 Markdown 拥有更灵活的布局、图形和交互性，且当前 LLM 已能很好支持。
- **输入端也需要改进**：当前的文本/音频/视频输入模态缺少指向、手势和空间参照。
- **长期方向**：由扩散模型生成的交互式神经视频/模拟，融合程序化（Software 1.0）产物与神经产物。
- **社区验证**：多位从业者报告已在生产环境中成功使用基于 HTML 的 LLM 输出。

---

## 详细内容

### 格式层级

Karpathy 提出了一个清晰的 LLM 输出丰富度层级：

1. **纯文本** — 阅读费力，结构最少。
2. **Markdown** — 当前默认格式，加粗、斜体、标题、表格带来一些视觉改善。
3. **HTML** — 底层仍是代码，但在布局、图形和交互性上远比 Markdown 灵活，正在成为新的默认选择。
4. **交互式神经视频/模拟** — 演进的终点：AI 通过扩散网络直接生成交互式视觉媒体。

核心洞察：每一层都更好地匹配人脑的视觉处理能力，降低理解 AI 输出的认知负担。

### 实践技巧：请求 HTML 输出

一个可立即使用的技巧：在查询 LLM 时，在提示词末尾加上 "structure your response as HTML"。社区反馈证实这在生产环境中效果良好：
- ScribaAI 已将此方法用于商业产品，表示"让 LLM 自己写渲染逻辑省掉了一整条流水线"。
- 主要失败模式是移动端布局崩坏，在提示词中加一个示例即可解决。
- HTML 保留了 Markdown 丢失的结构：表格还是表格，嵌套列表还是嵌套列表。

### 输入端的差距

Karpathy 同时指出输入侧也需要演进。当前模态（音频、文本、视频）缺少在屏幕上指向、手势、空间参照的能力——而这些是坐在电脑旁边和人交流时自然会用到的。这表明多模态输入（超越语音和文字）是一个重要前沿方向。
