## Summary / 摘要

Andrej Karpathy argues that LLM output should evolve beyond markdown toward HTML and eventually interactive neural media. Vision is the brain's highest-bandwidth input channel (~1/3 of cortical processing), so AI outputs should increasingly leverage visual richness. A practical tip: append "structure your response as HTML" to LLM prompts for immediate quality gains.

Andrej Karpathy 认为 LLM 输出应从 Markdown 演进到 HTML，最终走向交互式神经媒体。视觉是大脑带宽最高的输入通道（约占皮层处理的 1/3），因此 AI 输出应越来越多地利用视觉丰富性。一个实用技巧：在 LLM 提示词末尾加上 "structure your response as HTML" 即可获得立竿见影的效果提升。

## Key Points / 要点

- **Output format progression**: raw text → markdown → HTML → interactive neural videos. Each step increases information bandwidth to the human brain.
- **Vision as primary output channel**: ~1/3 of the human brain is dedicated to visual processing; AI should exploit this "10-lane superhighway."
- **HTML as the practical next step**: More flexible layout, graphics, and interactivity than markdown. Already works well with current LLMs.
- **Input side also needs improvement**: Pointing, gesturing, and spatial referencing are missing from current text/audio/video input modalities.
- **Long-term direction**: Interactive neural videos/simulations generated by diffusion models, blending procedural (Software 1.0) artifacts with neural artifacts.
- **Community validation**: Multiple practitioners report shipping HTML-based LLM output in production successfully.

- **输出格式递进**：纯文本 → Markdown → HTML → 交互式神经视频，每一步都提升了向人脑传递信息的带宽。
- **视觉是主要输出通道**：人脑约 1/3 专门处理视觉信息；AI 应充分利用这条"10 车道高速公路"。
- **HTML 是实际可行的下一步**：比 Markdown 拥有更灵活的布局、图形和交互性，且当前 LLM 已能很好支持。
- **输入端也需要改进**：当前的文本/音频/视频输入模态缺少指向、手势和空间参照。
- **长期方向**：由扩散模型生成的交互式神经视频/模拟，融合程序化（Software 1.0）产物与神经产物。
- **社区验证**：多位从业者报告已在生产环境中成功使用基于 HTML 的 LLM 输出。

## Details / 详细内容

### The Format Hierarchy / 格式层级

Karpathy outlines a clear hierarchy of LLM output richness:

1. **Raw text** — Effortful to read, minimal structure.
2. **Markdown** — Current default. Bold, italic, headings, tables add some visual relief.
3. **HTML** — Procedural underneath, but far more flexibility for layout, graphics, and interactivity. Early but forming as a new good default.
4. **Interactive neural videos/simulations** — The extrapolation endpoint: AI generates interactive visual media directly via diffusion networks.

The key insight is that each level better matches the brain's visual processing capacity, reducing the cognitive effort required to parse AI output.

### Practical Application: Asking for HTML / 实践：请求 HTML 输出

A concrete, immediately actionable technique: when querying an LLM, append *"structure your response as HTML"* to the prompt. Community feedback confirms this works well in production:
- ScribaAI ships this approach commercially, noting that "letting the LLM write its own rendering saved us a real pipeline."
- The main failure mode is layout breaking on mobile; one example in the prompt fixes this.
- HTML preserves structure that markdown loses: tables stay tables, nested lists stay nested.

### The Input Gap / 输入端的差距

Karpathy also highlights that the input side needs evolution. Current modalities (audio, text, video) lack the ability to point, gesture, or spatially reference things on screen — the kind of interactions that come naturally when sitting next to someone at a computer. This suggests multimodal input beyond just voice and text is an important frontier.
